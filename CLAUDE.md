# Data Processing

Reference data-processing pipelines and examples for Open Data Hub / Red Hat OpenShift AI.
Focuses on document conversion and chunking using Docling, packaged as Kubeflow Pipelines (KFP), Jupyter Notebooks, and helper scripts.

Repository: [opendatahub-io/data-processing](https://github.com/opendatahub-io/data-processing)

## Architecture

```
PDF/Documents → Import (S3/HTTP) → Split → Docling Convert → Chunk (optional) → JSONL output
```

**Core flow**: KFP pipelines import PDFs, split them for parallel processing, convert via Docling (standard OCR or VLM), and optionally chunk output using HybridChunker for RAG.

Two pipeline variants:
- **Standard** (`docling-standard/`): Traditional Docling models (layout + tableformer)
- **VLM** (`docling-vlm/`): Vision-language models (SmolVLM/SmolDocling), supports local or remote inference

Shared KFP components (`common/components.py`): `import_pdfs`, `create_pdf_splits`, `download_docling_models`, `docling_chunk`

Granular targets:
```bash

```

Run directly with pytest:
```bash
pytest                           # Run all tests (uses pyproject.toml config)
pytest tests/unit/               # Unit tests only (by directory)
pytest tests/integration/        # Integration tests only (by directory)
pytest -m unit                   # Unit tests only (by marker — equivalent)

```

### Test Isolation

Tests are isolated by **directory structure**:

```
tests/
  unit/           <- fast, no external deps (auto-marked @pytest.mark.unit)
  integration/    <- slow, needs GPU/network/Docker (auto-marked @pytest.mark.integration)
  fixtures/       <- shared sample data
```

Each subdirectory has its own `conftest.py` that auto-applies the appropriate pytest marker. To add a new test, place it in the correct directory — no manual marker annotation needed.

Markers (`unit`, `integration`) are also registered in `pyproject.toml` and can be used with `pytest -m unit` as an alternative to directory-based selection.

### Coverage

Configured via `pytest-cov` in `pyproject.toml` under `[tool.coverage.*]`. Coverage tracks `scripts/subset_selection/` and `kubeflow-pipelines/common/`. CI uploads reports to Codecov on PRs. Minimum threshold: 40%.

```bash
make coverage                    # Terminal + HTML report (htmlcov/)
pytest -m unit --cov             # Quick terminal summary
```

### Test Details

- **Notebook parameter tests**: Every notebook must have a code cell tagged `parameters` for papermill injection
- **Notebook execution tests**: Runs each notebook end-to-end via papermill with test parameters defined in `tests/integration/test_notebook_execution.py:19-41`
- **Skippable notebooks**: Configure in `tests/conftest.py` via `SKIP_FOR_NOW` list
- **Pipeline compile tests**: CI compiles pipeline Python and diffs against committed YAML

## Formatting, Linting & Type Checking

Handled by pre-commit hooks (`.pre-commit-config.yaml`). Install with `pre-commit install && pre-commit install --hook-type pre-push`.

**On commit** (fast, <5s):
- **ruff-lint**: Linting with autofixes (rules: E, F, I, B, W, UP)
- **ruff-format**: Code formatting (line-length 88, target Python 3.12)
- **nbstripout**: Strips notebook outputs
- **check-yaml**: Validates YAML syntax (workflows, pipeline configs)
- **end-of-file-fixer**, **trailing-whitespace**, **check-merge-conflict**: Hygiene
- **check-added-large-files**: Blocks files >500KB

**On push** (slower, ~15s):
- **mypy**: Type checking across `scripts/`, `tests/`, `kubeflow-pipelines/`

Manual commands:
```bash
make format-python       # Auto-format Python files
make format-notebooks    # Strip notebook outputs
make lint                # Run all lint/format checks
make typecheck           # Run mypy type checks
```

Ruff config: `pyproject.toml` (rules: E, F, I, B, W, UP)

### Type Checking (mypy)

Configured in `pyproject.toml` under `[tool.mypy]`. Key settings:
- `check_untyped_defs = true` — checks function bodies even without annotations
- `disallow_incomplete_defs = true` — requires all params to be annotated if any are
- `no_implicit_optional = true` — requires explicit `Optional[X]` instead of `X = None`
- `warn_return_any = true` — flags functions that return `Any` without annotation

KFP components have relaxed import checking (`disable_error_code = ["import-not-found"]`) because they use inline imports for packages only available in container base images. Third-party libraries without stubs (kfp, docling, h5py, etc.) have `ignore_missing_imports = true`.

## Debugging Guide

**Pipeline compile failures**: Regenerate YAML and commit it — see "Generated Code" section above.

**Notebook test failures**:
- Check `tests/conftest.py` for skip list (`SKIP_FOR_NOW`)
- Check test parameters in `tests/integration/test_notebook_execution.py:23-41` for notebook-specific overrides
- Subset selection notebook requires GPU; use `testing_mode=True` for CPU fallback

**KFP component imports**: Components use `pylint: disable=import-outside-toplevel` because KFP serializes component functions — imports must be inside the function body.

**Local pipeline testing**: Each pipeline has a `local_run.py` for Docker-based local execution without a Kubernetes cluster.
